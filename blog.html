Academic Reflections

From Syariah Laws to Software Code: A Modern Transition

Transitioning from a Syariah background at Form 6 (SMK Baling) to Software Engineering at UniSZA taught me an unexpected lesson: legal code and computer code are both built on strict structural logic. Read about how parsing legal texts prepared my mind for algorithmic troubleshooting...

When I studied in Form 6 at SMK Baling taking the Syariah Course, I spent my days reading constitutional structures, conditional clauses, and logical compliance requirements. I never expected that this background would become my greatest asset when transitioning into software development at UniSZA (FIK).

In Syariah studies, you learn to analyze a core text rule, determine dependencies, and look for exceptions based on specific parameters. Programming follows the exact same logic. When writing an if/else branch loop statement in JavaScript, I am performing the exact same structural analysis I practiced in Baling, just translated into code.

Key takeaways from this interdisciplinary switch:

  • Rule-Based Reasoning: Systems architectures run smoothly when conditions are defined properly.
  • Problem-Solving Focus: Troubleshooting an application bug feels identical to breaking down a complex institutional case study.
  • Unique Value: Combining humanities and strict technical logic allows me to look at UI/UX design with fresh, user-centric perspectives.

Demystifying CSD 34203: Building My Portfolio Site

A comprehensive walkthrough detailing how I developed this responsive static blog platform. From structuring modular semantic layouts with pure HTML5/CSS3 to managing multi-commit version controls using clean repository structures on GitHub.

Developing this personal portfolio as part of my CSD 34203 Special Topics in Software Development coursework required merging clean visual aesthetics with high-performance execution.

Instead of relying on bloated, heavy external graphical styling templates, I structured this platform from scratch using semantic HTML5 layouts, advanced CSS Root variable systems for dark/light themes, and light vanilla JavaScript engines.

Technical Milestone Highlights:

  • Clean Repository Management: Maintained an organized workspace structure on GitHub with concise, descriptive tracking messages across commits.
  • Fluid Responsiveness: Built flexible UI viewports using CSS Grid systems and Media Queries to ensure flawless readability across phones, tablets, and desktop setups.
  • No Purple Backgrounds: Replaced distracting gradient banners with minimal, dark-mode compliant accents to maximize professional presentation value.
